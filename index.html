CoronaTracker

This projects primarily serves as a simple tracker for the number of confirmed cases of Coronavirus in China. Secondary goal is to evaluate models generated by curve fitting problems.

Graph

Models

It is commonly suggested that a virus spreads in an exponential way until artificial immunization is available (Wikipedia). Hence curve fittings problems are solved to test if this hypothesis applies to 2019-nCoV as well.


Linear Models

Sadly, this situation cannot fully modeled by a linear curve fitting problem of the form f(x) = a*e^x+b*x+c where the factors a, b, c are choosen so that the model's distance to each real datapoint is as minimal as possible. This is essentially because the most important aspect of the current situation is the effective reproduction rate of the virus that is somehow contained in the base of the exponential term.

Due to limited time and resources the first attempt to model the situation used a linear model despite its described shortcomings. With guesswork it was found that an acceptable base is e/2=1.35.

g(t) = a*(e/2)^t+bt+c

Nonlinear Models

However, thanks to additional work (and Python libraries), nonlinear optimization can be used to train models.

Because pure exponential functions started to perform worse each day, a new class of nonlinear equations is introduced:

h(t) = a*(1+tanh(s*t-v))

This method's parameter have the following meanings:
  • a: Amplitude (half of the maximum value of h(t) when t is infinite)
  • s: Slope (how fast the model reaches 2*a)
  • v: Movement to the left/right relative to t=0

Sadly, this model can't be automatically trained in your browser, hence the values are manually updated (values in brackets show past values with newest value left):

  • a: 12878 (10882, 9313, 9141)
  • s: 0.194 (0.209, 0.224, 0.225)
  • v: 4.52 (4.71, 4.87, 4.89)
  • Root Mean Square Error (RMSE): 160.14 (110.80, 74.19, 76.18)

In addition, there is now also an exponential model of which the base can be optimized. Despite the more advanced mathematical concepts behind this model (compared to g(t)) this model doesn't perform meaningfully better.

i(t) = a*b^(t-v)

This method's parameter have the following meanings:
  • a: Amplitude
  • b: Base of the exponential function
  • v: Movement to the left/right

Sadly, this model can't be automatically trained in your browser, hence the values are manually updated (values in brackets show past values with newest value left):

  • a: 112.36 (18.42, 21.75)
  • b: 1.27 (1.30, 1.33)
  • v: 4.20 (-1.45, 0.84)
  • Root Mean Square Error (RMSE): 585.16 (474.34, 368.36)

Interpreting the models

First things first: This models aren't created or used by epidemologists but me. For best information about the situation look at data from WHO or your local government.

However, I think these models provide some basic insights and need to be taken with a grain of salt:

  • During the first phase of such a situation, an exponential growth is a valid model. However, soon after countermeasures kick in, the exponential growth isn't as obvious.
  • The main reason why we were able to observe such "good" exponential growth is because the virus spread in a city. Until counter-measures this is a great approximation of an infinite number of people who can become infected (which is necessary for exponential growth).
  • We shouldn't panic. While these models can't predict the future, they show that there is a wide variety of possible outcomes from the current data, a continuing exponential growth of case numbers is just one of them.

Overall conclusion: Exponential growth of case numbers during the first phase of a virus isn't a reason to panic. In fact during the current situation, there is some indication that the exponential phase might be over and we now see a sigmoid function. However, I wouldn't consider the existing data enough to make meaningful conclusions in any direction.

Background

How This Project Came To Be

I simply was interested in seeing if the Coronavirus spreaded in an exponential way. But as I couldn't find any graphical data, I decided to build my own dataset and publish it on Twitter. However, I assumue my followers aren't as interested in this graphs as I am, hence I decided to rather build this small personal project.

Can You Contribute?

Of course! Feel free to write a Pull Request on Github if you have newer or better data. Simply insert the data in the corresponding table and most importantly include a relieable source.

Sources must be from English or German media organizations.

If you wish to add functionality to this site, feel free as well.
